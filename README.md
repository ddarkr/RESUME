## 🔖 philosophy

- **나중을 생각하는 개발**
  - 프로젝트를 진행하다보면, 항상 나중에는 그 프로젝트를 유지보수하기 마련입니다. 작게는 버그 수정부터, 크게 본다면 기능이 추가될 수도 있습니다. 항상 개발 시에는 미래의 개발을 생각하여 정형화된 구조화, 컴포넌트의 재사용성, 코드의 가독성 등을 신경써서 개발하도록 노력합니다.  
- **사용자 중점 기획**
  - 사용자의 피드백을 수용하고 개선하며 나아가길 희망합니다. 서비스를 사용하는 사용자의 기준에서 생각해보고 개선점이 있을까 찾아봅니다.
  - [애자일 소프트웨어 개발 선언](https://agilemanifesto.org/iso/ko/manifesto.html)
- **배움의 끝은 없다**
  - 새로운 기술이나 서비스, 개발방법론을 관심 있게 찾아보며 프로젝트에 적용을 생각해봅니다.
